The Horizon: Multi-Cloud & Egress Economics

While the MVP is built on Cloudflare, the didbox402 protocol is cloud-agnostic. To support true decentralization, the infrastructure must be portable across major cloud providers.

The Egress Challenge

Cloudflare R2 is unique in its zero-egress model. Moving to AWS (S3), GCP (GCS), or Azure introduces bandwidth costs that can exceed the cost of storage.

The Edge Observation: On traditional clouds, "Retrieve" is more expensive than "Store."

Cloudflare (R2)

Pricing: Purely duration + size.

Advantage: Symmetrical costs. High frequency agent talk is cheap.

Standard Cloud (S3/GCS)

Pricing: Storage + High Egress.

Protocol Shift: Retrieval must be paid (x402) per download to cover bandwidth.

Adaptive Pricing

To support multi-cloud, the node operator configures Inbound (Store) and Outbound (Retrieve) rates independently. This ensures nodes on traditional clouds remain sustainable.

Portability Strategy

  1. S3 Compatibility: Use S3-compatible APIs for blob operations.
  2. Relational Metadata: Use SQL (Postgres/MySQL) for lease metadata.
  3. Compute: Hono runs on Node.js, Deno, and Bun for easy cross-cloud deployment.
