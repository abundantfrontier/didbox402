Extension Points v0.6.2

The didbox402 protocol is designed to be modular and extensible. Commercial providers can customize their nodes at several key integration points.

1. Payment Rails

While the reference node supports L402 and x402, you can add any rail that follows the 402 handshake pattern.

  • Stablecoins: Add Solana/USDC or Arbitrum/USDC.
  • Fiat: Use Stripe for credit card authorizations (Pre-paid credits).
  • LSP Integration: Connect to LND, CLN, or Alby Hub.

2. Storage Backends

The STORAGE_BUCKET interface can be implemented for any object store.

  • Cloud: AWS S3, Google Cloud Storage, Azure Blob.
  • Decentralized: IPFS gateways, Arweave, Filecoin.
  • Local: MinIO or standard filesystem storage.

3. Custom Authorization

Beyond DID signatures, you can implement richer access control.

  • Verifiable Credentials (VCs): Require an uploader to prove they are a member of a specific DAO.
  • ZKP Proofs: Allow anonymous retrieval based on zero-knowledge membership proofs.

4. Pricing Models

The pricing logic is localized in lib/pricing.ts and can be overridden.

  • Tiered Pricing: Volume-based discounts for large agents.
  • Subscription: Allow DIDs with a specific NFT or token to access storage for free.
