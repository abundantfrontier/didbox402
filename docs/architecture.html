The Engine: Architecture

didbox402 is built on the Cloudflare Edge. It utilizes serverless primitives to ensure that the storage is as close to the agents as possible.

Component Stack

  • Hono: Routing and Middleware.
  • R2: Blob Storage (Opaque ciphertext).
  • D1: Metadata (Ownership and Expiration).

The "Box" Lifecycle

  1. Lease: Client posts ciphertext + x402 payment.
  2. Live: Authorized DIDs retrieve data until expiration.
  3. Extend: Payment adds more time to the lease.
  4. Purge: Expired data is automatically deleted.
