The Shield: Threat Model v0.5.0

didbox402 v0.5.0 significantly hardens the protocol against interception and replay attacks.

1. Metadata Leakage (The Social Graph)

Threat: The storage provider can see who is communicating with whom by observing salted DID hashes.

Mitigation: Mandatory Salted Hashing for all indexes (Inboxes, Storage Records). Provider never sees raw DIDs.

2. Payload Size & Frequency (Traffic Analysis)

Threat: Even if data is encrypted, the size of the ciphertext and retrieval frequency can leak the payload type.

Mitigation: Implementation of client-side Padding to normalize traffic profiles.

3. Replay Attacks

Threat: An attacker intercepts a signed storage request and resubmits it to drain funds or overwrite data.

v0.5.0 Mitigation: Mandatory Nonce Tracking and Temporal Binding. Signatures are single-use and expire after 5 minutes.

4. Infrastructure DoS (Janitor)

Threat: Malicious actors trigger the Janitor endpoint to perform expensive cleanup operations or disrupt service.

v0.5.0 Mitigation: Mandatory X-Admin-Token requirement for all Janitor operations.

Trust Assumptions

  • Client Integrity: We assume the client handles their Ed25519 keys securely.
  • Infrastructure: We trust the underlying cloud (R2/S3) to maintain bit-integrity.
