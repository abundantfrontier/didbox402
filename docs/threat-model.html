The Shield: Threat Model v0.7.0 (Normative Reference)

Honesty starts with admitting what didbox402 cannot protect. While we aim for maximum privacy, the physical reality of hosting and networking creates metadata "exhaust."

1. Key Assets & Trust Boundaries

Asset Trust Boundary Protection
Plaintext Data Client-Side End-to-End Encryption (E2EE)
Identity (DID) Protocol-Wide Salted Hashing & Ed25519 Signatures
Funds (Sats/USDC) Wallet-Side 402 Challenge-Response Handshake

2. Main Threat Categories

Authentication (The Shield)

Threat: An attacker intercepts a signed storage request and resubmits it to drain funds or overwrite data.

Mitigation: Mandatory Nonce Tracking and Temporal Request Hash binding. Signatures are single-use and valid for ±5 minutes.

Payments (The Rail)

Threat: A provider accepts payment but refuses retrieval or deletes data early.

Mitigation: didbox402 is a Commodity Protocol. Agents should diversify storage across multiple nodes. Provider reputation acts as a market signal.

Privacy (The Shadow)

Threat: The storage provider observes who is communicating with whom by observing DID hashes.

Mitigation: Salted DID Hashing for all lookup indexes. Observers cannot link storage boxes to recipients without knowing the secret node salt.

Availability (Infrastructure)

Threat: Malicious actors trigger the Janitor endpoint to disrupt service.

Mitigation: Mandatory X-Admin-Token requirement for all Janitor operations.

Sovereign Mobility (v0.7.0+)

Threat: A malicious source node issues fake Migration Proofs or a node publishes a fraudulent node_identity.

Mitigation: Migration Proofs are only useful for the legitimate owner. Destination nodes are not required to trust them in Phase 1. Node identity is published in discovery and can be validated by clients.

3. Trust Assumptions

  • Client Integrity: We assume the client handles their Ed25519 keys securely and never shares the plaintext.
  • Infrastructure Bit-Integrity: We trust the underlying cloud (R2/S3) to maintain data for the lease duration.
