The Horizon: Multi-Cloud & Egress Economics

While the MVP is built on Cloudflare, the didbox402 protocol is cloud-agnostic. To support true decentralization and agent autonomy, the infrastructure must be portable across major cloud providers.

The Egress Challenge

Cloudflare R2 is unique in its zero-egress model. Moving to AWS (S3), GCP (GCS), or Azure (Blob) introduces bandwidth costs that can exceed the cost of storage itself.

The Edge Observation: On traditional clouds, "Retrieve" is more expensive than "Store."

Cloudflare (R2)

Pricing: Purely duration + size.

Advantage: Symmetrical costs. High frequency agent-to-agent talk is cheap.

Standard Cloud (S3/GCS)

Pricing: Storage + High Egress.

Protocol Shift: Retrieval must be paid (x402) per download to cover bandwidth.

Adaptive Pricing Model

To support multi-cloud, the calculatePrice logic must be modular:

  • Inbound (Store): Covers the cost of disk space for the lease duration.
  • Outbound (Retrieve): Covers the provider's egress tax. In the MVP, this is 0. In AWS/GCP nodes, this becomes a required x402 payment for the GET request.

Portability Strategy

  1. Abstraction Layer: Use the S3-compatible API for all blob operations.
  2. Metadata Agnosticism: While we use D1 (SQLite), any relational DB (Postgres/RDS) can store the lease metadata.
  3. Compute Portability: Hono runs on Node.js, Deno, and Bun, making it easy to deploy on Lambda, Fly.io, or Google Cloud Run.
