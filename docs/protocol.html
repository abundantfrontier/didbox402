The Handshake: Protocol Spec v0.3.0

Formal Specification: PROTOCOL.md (v0.3.0)

Interaction with didbox402 is stateless and authenticated via DID-signed HTTP headers.

Required Headers

Header Description Example
X-DID Multibase Ed25519 did:key identifier. did:key:z6Mkp...
X-DID-Signature Hex-encoded Ed25519 signature of the request hash. abcd123...
X-DID-Timestamp Unix Epoch in milliseconds (UTC). Drift window: ±5m. 1715436000000
X-Payment The x402 payment proof (preimage). preimage:...
X-Inbox-Alias (Optional) Target scoped inbox alias. project-x

Replay Protection (Nonce Tracking)

didbox402 implements mandatory Nonce Tracking. The server caches every X-DID-Signature within the 5-minute window. Any attempt to re-use a valid signature will be rejected with 401 Unauthorized.

Signature Binding

To prevent tampering, the signature MUST cover the Temporal Request Hash:

Hash = SHA256(Timestamp + Method + Path + SHA256(Raw_Body))

In the reference SDK, this is generated as follows:

const signature = await ed.sign(
  sha256(`${timestamp}${method}${path}${bodyHash}`),
  privateKey
);
