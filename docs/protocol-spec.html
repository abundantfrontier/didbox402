Formal Protocol Specification v0.6.2

didbox402 is an agent-native protocol for ephemeral, paid, and verifiable storage. It facilitates trustless data handoff between autonomous entities using Decentralized Identifiers (DIDs) and micropayments.

1. Introduction

1.1 Design Goals

  • Autonomy: Agents lease storage without human intervention or accounts.
  • Privacy: Metadata obfuscation via salted hashing and E2EE.
  • Scalability: Stateless, edge-native architecture.
  • Verifiability: Every request is signed and economically verified.

2. Authentication (The Shield)

Every request MUST be authenticated via DID signatures with temporal binding.

2.1 Headers

  • X-DID: Multibase Ed25519 identifier (z6Mk...).
  • X-DID-Signature: Hex-encoded EdDSA signature.
  • X-DID-Timestamp: Unix Epoch in milliseconds.

2.2 Clock Skew

Servers MUST reject requests with a timestamp drift > 5 minutes. Servers MUST include a Date header to assist clients in synchronization.

3. Economics (The Rail)

Resource allocation uses the 402 Payment Required standard over Lightning (L402) or Web3 (x402).

3.1 Dynamic Pricing

Total_Cost = Math.max(1MB, Size) * Duration_Hours * Base_Rate

3.2 x402 Settlement

Proof of payment is provided via the PAYMENT-SIGNATURE header containing the transaction hash of a successful USDC transfer to the node's address.

4. API Specification

4.1 POST /store

Leases a new storage box.

Request Body: { ciphertext, durationHours, recipientDid?, inboxAlias? }
Response: { storageId, expiresAt, sizeBytes, pricePaidSatoshis }

4.2 GET /retrieve/{id}

Retrieves box ciphertext.

Response: { ciphertext }

4.3 GET /inbox/{alias}

Lists active boxes for the authenticated DID in the specified alias.

Response: { alias, items: [{ id, sizeBytes, expiresAt }] }

4.4 GET /leases

Lists all active leases created by the authenticated DID.

Response: { leases: [{ id, sizeBytes, expiresAt, recipientHash }] }

4.5 GET /price

Discover current rates.

Response: { base_rate_per_mb_hour, inbox_creation_fee, ... }
