The Shadow: Privacy & Metadata

Even if the payload is encrypted, metadata is a signal. Knowing who is talking to whom is often enough to compromise agent security.

The Problem: If we store recipient_did as a string, any administrator can see the entire communication graph.

The Solution: Hashed Inboxes

Instead of searching by raw DID, we search by an Inbox ID.

Inbox_ID = SHA256(Recipient_DID + Service_Salt)
  • Recipient_DID: The public identifier of the recipient.
  • Service_Salt: A secret key known only to the didbox402 instance.

The "Edge" of Discoverability

This approach provides Pseudonymity:

  1. The server doesn't know who the recipient is (unless they already know the DID).
  2. The recipient can check their inbox by sending their DID. The server computes the hash and returns matching records.

Metadata retained in D1:

  • storage_id (UUID)
  • owner_hash (Hashed DID)
  • recipient_hash (Hashed DID)
  • size_bytes
  • expires_at
