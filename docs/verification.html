The Proof: Verification & Conformance

A protocol is only as good as its verification. This document defines the mandatory test cases that any didbox402 implementation must pass to be considered compliant.

1. Economic Integrity (Pricing)

Goal: Ensure the server calculates pricing exactly as specified in the formula.

Verification: Send a POST /store request with a known size and duration but no payment. The 402 Payment Required response must return the exact expected Satoshi amount.

EXPECT(resp.amount_satoshis).TO_EQUAL(Math.ceil((size_mb * duration_hours) * base_rate))

2. Cryptographic Isolation (Inboxes)

Goal: Prove that knowing one inbox alias reveals nothing about another.

Verification: Store an item in alias-a. Query GET /inbox/alias-b. The response must be empty, even if the same DID owns both.

EXPECT(GET("/inbox/alias-b").items).TO_BE_EMPTY()

3. Temporal Persistence (Expiration)

Goal: Confirm that data is unreachable precisely after expiration.

Verification: Store an item with durationHours: 0 (or a very short lease). Wait for expiration. Attempt GET /retrieve/{id}.

EXPECT(GET("/retrieve/id").status).TO_EQUAL(410) // Gone/Expired

4. Sovereign Access (Authorization)

Goal: Ensure only the owner or intended recipient can retrieve data.

Verification: Attempt to retrieve an object using a DID that is neither the owner nor the recipient for the scoped inbox.

EXPECT(GET("/retrieve/id", headers={"X-DID": "evil-did"}).status).TO_EQUAL(403)

Reference Implementation Tests

The didbox402 repository includes a complete reference test suite that implements these conformance checks using Vitest and Cloudflare Workers Vitest Pool.

# Run all conformance tests
npm test

Coverage Summary

  • Unit Tests: Pricing logic (MB-hour accuracy), DID hashing, and storage helpers.
  • Integration Tests: Full HTTP request/response cycles against a simulated edge environment (Miniflare).
  • Economic Tests: Verification of 402 rejections and correct Satoshi calculations.

Developers implementing the protocol on other clouds (AWS/GCP) should mirror these test cases to ensure cross-node compatibility and agent trust.
