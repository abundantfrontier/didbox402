The Proof: Verification & Conformance

A protocol is only as good as its verification. This document defines the conformance test cases that any didbox402 implementation must pass to be considered compliant.

Mandatory Core Tests

1. Economic Integrity

Verification: POST /store without payment must reject with 402 and the correct Satoshi amount.

EXPECT(resp.status).TO_EQUAL(402)
EXPECT(resp.amount_satoshis).TO_EQUAL(Math.ceil(size_mb * duration * rate))

2. Cryptographic Isolation

Verification: Querying a named inbox for data stored in a different alias must return empty results.

EXPECT(GET("/inbox/alias-b").items).TO_BE_EMPTY()

3. Sovereign Access

Verification: Only the owner or recipient (scoped by alias) can retrieve the ciphertext.

EXPECT(GET("/retrieve/id", headers={"X-DID": "wrong-did"}).status).TO_EQUAL(403)

4. Replay Protection

Verification: Modifying the request body after signing must invalidate the request.

EXPECT(res.status).TO_EQUAL(401) // Signature Binding Mismatch

Reference Conformance Suite

The didbox402 repository includes an automated test suite that enforces these rules. It also provides a Client Library that implements the correct signature binding and payment flows.
