The Proof: Verification & Conformance

A protocol is only as good as its verification. This document defines the conformance test cases that any didbox402 implementation must pass to be considered compliant, allowing agents to automatically verify provider honesty.

Mandatory Core Tests

1. Economic Integrity (Pricing)

Goal: Ensure the server calculates pricing exactly as specified in the formula, refusing service without payment.

Verification: Send a POST /store request with a known size and duration but omit the X-Payment header.

POST /store HTTP/1.1
X-DID: did:key:z6Mkp...
X-DID-Signature: mock_sig

{
  "ciphertext": "hello world",
  "durationHours": 1
}

Expected Response:

HTTP/1.1 402 Payment Required
{
  "error": "Payment Required",
  "amount_satoshis": 100
}
EXPECT(resp.amount_satoshis).TO_EQUAL(Math.ceil((size_mb * duration_hours) * base_rate))

2. Cryptographic Isolation (Inboxes)

Goal: Prove that knowing one inbox alias reveals nothing about another, even for the same DID.

Verification: Store an item in alias-a. Then, query GET /inbox/alias-b.

GET /inbox/alias-b HTTP/1.1
X-DID: did:key:z6Mkp...
X-DID-Signature: mock_sig

Expected Response:

HTTP/1.1 200 OK
{
  "alias": "alias-b",
  "items": []
}
EXPECT(GET("/inbox/alias-b").items).TO_BE_EMPTY()

3. Temporal Persistence (Expiration)

Goal: Confirm that data is unreachable precisely after the leased time expires.

Verification: Store an item with durationHours: -1 (or wait for natural expiration). Attempt GET /retrieve/{id}.

GET /retrieve/uuid-1234 HTTP/1.1
X-DID: did:key:z6Mkp...
X-DID-Signature: mock_sig

Expected Response:

HTTP/1.1 410 Gone
{
  "error": "Expired"
}
EXPECT(GET("/retrieve/id").status).TO_EQUAL(410)

4. Sovereign Access (Authorization)

Goal: Ensure only the owner or intended recipient can retrieve data.

Verification: Attempt to retrieve an object using a DID that is neither the owner nor the recipient.

GET /retrieve/uuid-1234 HTTP/1.1
X-DID: did:key:zEvil...
X-DID-Signature: evil_sig

Expected Response:

HTTP/1.1 403 Forbidden
{
  "error": "Unauthorized"
}
EXPECT(GET("/retrieve/id", headers={"X-DID": "evil-did"}).status).TO_EQUAL(403)

Recommended Tests

5. Replay Protection (Signature Binding)

Goal: Ensure an attacker cannot replay a valid signature on a modified request.

Verification: Intercept a valid POST /store request and alter the durationHours. Resubmit the request.

Expected Response: 401 Unauthorized (Signature mismatch).

6. Egress Asymmetry (Multi-Cloud)

Goal: On infrastructure like AWS/GCP, ensure retrieval incurs dynamic x402 cost.

Verification: Call GET /retrieve/{id} on a non-Cloudflare node without X-Payment.

Expected Response: 402 Payment Required with calculated egress Satoshis.

Running the Reference Suite

The didbox402 repository includes a complete reference test suite implementing the mandatory core checks against a simulated edge environment.

# The reference suite is located at:
src/__tests__/index.test.ts

# Run all conformance tests locally:
npm test

Developers implementing the protocol on other clouds or language runtimes should mirror these test cases to ensure cross-node compatibility and agent trust.
