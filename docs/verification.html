The Proof: Verification & Conformance v0.4.0

A protocol is only as good as its verification. Version v0.4.0 introduces the Testing Gauntlet: a suite of 21 automated tests verifying security, privacy, and dual-rail economic compliance.

Mandatory Core Tests

1. Economic Integrity

Verification: POST /store without payment must reject with 402 and the correct Satoshi amount.

EXPECT(resp.status).TO_EQUAL(402)
EXPECT(resp.amount_satoshis).TO_EQUAL(Math.ceil(size_mb * duration * rate))

2. Cryptographic Isolation

Verification: Querying a named inbox for data stored in a different alias must return empty results.

EXPECT(GET("/inbox/alias-b").items).TO_BE_EMPTY()

3. Sovereign Access

Verification: Only the owner or recipient (scoped by alias) can retrieve the ciphertext. Evil DIDs are blocked.

EXPECT(GET("/retrieve/id", headers={"X-DID": "wrong-did"}).status).TO_EQUAL(403)

4. Replay Protection (Nonce Tracking)

Verification: Re-using a valid signature with the same timestamp must be rejected immediately.

EXPECT(res2.status).TO_EQUAL(401) // Signature Already Used

5. Temporal Security (Drift Window)

Verification: Requests with an X-DID-Timestamp more than 5 minutes in the past or future must be rejected.

EXPECT(GET("/retrieve/id", headers={"X-DID-Timestamp": now - 301000}).status).TO_EQUAL(401) // 5m 1s Drift Rejected

6. Privacy Invariant

Verification: Prove that raw DIDs are never stored in the database.

EXPECT(DB_QUERY("SELECT * FROM storage_records")).NOT.TO_CONTAIN(MY_RAW_DID)

7. Rail Interoperability

Verification: The server must accept both L402 (Authorization) and x402 (X-Payment) proofs for the same resource.

EXPECT(GET("/retrieve/id", headers={"Authorization": "L402 ..."}).status).TO_EQUAL(200)
EXPECT(GET("/retrieve/id", headers={"X-Payment": "..."}).status).TO_EQUAL(200)

Reference Conformance Suite

The didbox402 repository includes 21 automated tests across four modular files:

  • auth.test.ts: Headers, Multibase parsing, Drift windows, and Janitor security.
  • storage.test.ts: Store/Retrieve flow, Nonce tracking, and Privacy invariants.
  • inboxes.test.ts: Alias creation, listing, and isolation.
  • index.test.ts: Economic handshake and 402 challenge cycles.
